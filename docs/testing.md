# 测试指南

## 测试文件总览

| 文件 | 类型 | 说明 |
|------|------|------|
| `test_all_apis.py` | API 集成测试 | 9 域 + 8 业务链，~150 测试用例 |
| `audit_test.py` | 业务审计测试 | 完整退租流程验证，2 样本 |
| `audit_full.py` | 全量审计测试 | 大规模审计 |
| `test_verification.py` | 验证测试 | 特定验证场景 |
| `backend/routes/` | 薄层路由 | 无独立测试（由 API 测试覆盖） |
| `backend/services/` | 业务层 | 可单独单元测试 |

## 运行测试

### 前置条件

```bash
# 1. 安装依赖
cd backend
pip install -r requirements.txt

# 2. 启动后端
python -m uvicorn main:app --host 0.0.0.0 --port 8005

# 3. 新终端运行测试
python test_all_apis.py
```

## 测试策略建议

```bash

python -c "
import requests
B='http://localhost:8005/api'; H={'X-API-Key':'dev-key-change-me'}
assert requests.get(B+'/health').status_code==200
assert requests.get(B+'/dashboard',headers=H).status_code==200

```

```bash
python test_all_apis.py
```

```bash
python audit_test.py
```

## 已知测试局限

1. **并发测试缺失**: 无多线程竞态条件测试（如同时创建合同、同时收款）
2. **大数据量测试缺失**: 无性能基准（如 1000 条账单查询）
3. **前端测试缺失**: 前端 Vue 组件无独立测试
4. **数据库恢复测试跳过**: 链 H-3 跳过（危险操作）
5. **文件上传测试缺失**: 图片上传/签名上传未完全覆盖
6. **Unicode/CJK 测试**: 中文编码在某些终端+curl 组合下可能解析异常（非代码问题）
