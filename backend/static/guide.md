## 1. 第一次打开系统

### 1.1 怎么打开（手机 App / 浏览器均可）

**如果你用的是手机 App（推荐）：**
直接点桌面上的「房东管家」图标，就像打开微信一样。

**如果你用的是电脑浏览器：**
打开浏览器，输入系统地址（问管理员要），看到「房东管理系统」几个字就说明成功了。

> 系统地址由管理员提供，通常类似 `https://landlord-xxx.onrender.com`。

### 1.2 怎么关闭

手机 App：直接划掉就行，数据自动保存。
浏览器：关掉网页即可。

## 2. 认识每个页面

打开系统后，底部有 5 个按钮，点哪个就去哪个页面。

### 2.1 首页（打开系统看到的第一个页面）

**你能看到什么：**
- 3 个大数字卡片：逾期账单数、空置房源数、待收租金
- 合同概览：按状态分类（待交房、已租、退租处理中）
- 快捷操作：一排按钮，点哪个就去哪个功能

**点击数字卡片会跳转到对应页面**。比如点「逾期账单」→ 进账单页看哪些欠费了。

**底部 5 个导航按钮：**

| 按钮 | 点进去干什么 |
|------|------------|
| 首页 | 看总体数据 |
| 房产 | 管理你的房子 |
| 租客 | 管理租客信息 |
| 合同 | 管理合同 |
| 账单 | 查看账单和收款 |

### 2.2 房产管理（底部点「房产」）

**这个页面干什么：** 管理你有几套房、每套房在哪、现在是空着还是租出去了。

**你能看到什么：**
- 一列房产卡片，每张上面写着：名字、地址、状态（空闲/已租/维修中）
- 顶部的下拉框：可以筛选只看「空闲」「已租」或「维修中」的房子

**你能做什么：**
- 点右上角 `+` → 新增房产
- 点某个房产卡片 → 编辑房产信息
- 在卡片上**手指左滑** → 出现红色「删除」按钮（只有没合同的空房子才能删）

**填写说明：**
- **房产名称**：给你的房子起个名，比如「尚景园1栋501」
- **地址**：详细地址，省市区+门牌号
- **状态**：默认「空闲」就行了，出租后自动变

### 2.3 租客管理（底部点「租客」）

**这个页面干什么：** 管理所有租过你房子的人。

**有两个标签页：**
- **在用**：现在正在租房的人
- **已退租**：以前租过、现在已经搬走的人（保留 150 天）

**你能做什么：**
- 在用标签：点右上角 `+` 新增租客，点租客名字看详细信息
- 已退租标签：可以搜索（输入姓名/手机号/身份证号），看到每个退租的人还剩多少天会被清理
- **恢复租客**：点进已退租的人，拉到最下面，点「恢复租客」按钮，这个人就重新变成「在用」状态了

**填写说明：**
- **身份证号**：必须 18 位（数字和字母 X），不能乱填
- **姓名**：真实姓名
- **手机号**：11 位，1 开头

### 2.4 合同管理（底部点「合同」）

**这个页面干什么：** 管理所有租赁合同。

**你能看到什么：**
- 一列合同卡片，显示：房子名、状态标签、合同编号、租客名、月租、押金、租期
- 顶部下拉框：可以按状态筛选（全部/待交房/已租/退租处理中）

**重要：退租完成的合同不会出现在这里**。结算确认后 1 分钟，合同自动从列表删除。想看历史合同数据，去「租客档案」页面。

**合同状态说明：**

| 状态 | 什么意思 |
|------|---------|
| 待交房 | 合同签了，但还没交房验收 |
| 已租 | 租客正在住 |
| 退租处理中 | 租客要退租了，正在办手续 |

**你能做什么：**
- 点右上角 `+` 新增合同
- 点某个合同卡片 → 进入合同详情页
- 在详情页可以：交房验收、看账单、看电表、发起退租、退房验收、结算退款

### 2.5 账单管理（底部点「账单」）

**这个页面干什么：** 查看每个月的账单，知道谁交了钱、谁没交。

**进入时自动补全**：打开账单页时，系统会自动检查——如果有某个月还没生成账单，会自动帮你生成。不需要手动操作。

**顶部显示：** 共 X 张账单 · X 张待付

**标签筛选：** 全部 / 未付 / 逾期 / 部分付 / 已付。点不同的标签看不同类型的账单。

**你能做什么：**
- 点某张账单 → 进入账单详情，看费用明细、收款记录
- 详情页可以：收款、催收
- 逾期账单会自动标红

### 2.6 收款管理（首页快捷操作 → 收款）

**这个页面干什么：** 记录租客每次交的钱。

**填写说明：**
- **收款日期**：哪天收的钱
- **金额**：收了多少钱（不能超过待付金额）
- **收款方式**：微信 / 支付宝 / 银行转账 / 现金
- **关联账单**：这笔钱是用来付哪张账单的

### 2.7 电表记录（合同详情 → 电表记录）

**这个页面干什么：** 记录每月电表读数，系统自动算电费。

**规则：**
- 每个合同**每月只能抄一次表**
- 电费自动算：`(这次读数 - 上次读数) × 1.2 元/度`
- 读数不能比上次小（不能倒退）

**顶部显示：** 一共几次抄表、累计用电多少度、总电费多少钱

## 3. 完整操作流程：从租客入住到退租

这是房东最常用的完整流程。跟着走一遍就全明白了。

### 3.1 第一步：登记房产

1. 点底部「房产」→ 点右上角 `+`
2. 填写房子名称和地址 → 点「创建」

### 3.2 第二步：登记租客

1. 点底部「租客」→ 点右上角 `+`
2. 填写身份证号、姓名、手机号 → 点「创建」

### 3.3 第三步：签合同

1. 点底部「合同」→ 点右上角 `+`
2. 选择刚才登记的房产和租客
3. 填写月租（比如 2500）、押金（通常 = 月租）
4. 交租日（每月几号交，1-28 之间）
5. 合同开始日期和结束日期（结束日期必须晚于开始日期）
6. 可以添加物品清单（床上、空调、冰箱等）
7. 点「创建合同」

### 3.4 第四步：交房验收

1. 点刚创建的合同卡片 → 进入合同详情
2. 点「交房验收」
3. 填写验收日期
4. **电表底数**：看电表上现在的数字，填进去。**这个非常重要！** 以后的电费都以这个为基础算
5. 点「确认验收」
6. 合同状态自动变成「已租」

### 3.5 第五步：每月抄表

1. 在合同详情页，点「电表记录」
2. 点「新增抄表」
3. 去电表上读当前数字，填进「当前读数」
4. 选抄表日期
5. 点「保存」→ 系统自动算电费

### 3.6 第六步：生成账单

1. 在合同详情页，点「查看账单」
2. 系统自动生成当月账单（打开账单页自动补全）
3. 进去后能看到：房租多少、水费多少、电费多少、一共多少钱

### 3.7 第七步：收款

1. 在账单列表里，点某张账单进去
2. 点「收款」
3. 填收款日期、金额、收款方式
4. 点「确认收款」
5. 如果收的金额 = 账单总额 → 状态变「已付」
6. 如果收的金额 < 账单总额 → 状态变「部分付」

### 3.8 第八步：退租

当租客要搬走时：

1. **发起退租**：合同详情页 → 点「发起退租」→ 确认
2. **抄最后一次电表**（如果当月还没抄）
3. **退房验收**：合同详情页 → 点「退房验收」
   - 填最后电表读数
   - 选钥匙是否归还（没归还要填扣款金额）
   - 逐项检查物品：完好 / 损坏（填扣款）/ 缺失（填扣款）
4. **生成结算单**：合同详情页 → 点「生成结算单」
   - 系统自动算：电费扣款 + 物品损坏扣款 + 物品缺失扣款 + 钥匙扣款 + 未付账单
   - 实际退款 = 押金 - 以上全部扣款
5. **确认退款**：填退款日期和方式 → 点「确认退款」
6. 合同 1 分钟后自动删除，房子变「空闲」，租客变「已退租」

### 3.9 退租后的数据去哪了

退租完成后，想查看历史数据：
1. 点底部「租客」→ 点「已退租」标签
2. 找到那个租客，点进去
3. 在租客档案里，展开折叠面板可以看到：
   - 合同物品清单
   - 交房验收记录
   - 退房验收记录（含每件物品的状态和扣款）
   - 每次抄表的记录
   - 每张账单
   - 每次收款
   - 结算单详情

**数据保留 150 天**，过期后需手动清理。

## 4. Excel 批量录入（最快上手方式）

如果你有多套房、多个租客要录入，**用 Excel 一次性导入是最快的方法**。比在网页上一条条填快 10 倍。

### 4.1 整体思路（三步走）

```
第一步：从系统下载模板（一个 Excel 文件）
第二步：在 Excel 里填写数据
第三步：把填好的 Excel 导入系统
```

### 4.2 第一步：下载模板

1. 打开系统首页（手机 App 点首页按钮，浏览器输入系统地址）
2. 找到快捷操作里的 **「备份数据」** 按钮，点击
3. 浏览器会下载一个文件，叫 `landlord_backup_XXXX.xlsx`
4. 用 Excel 或 WPS 打开这个文件

> **小技巧**：如果系统里已经有测试数据，下载的表格里就有示例数据，照着改就行了。

### 4.3 第二步：填写 Excel（逐表详解）

打开 Excel 后，底部有一排标签（叫 Sheet），每个 Sheet 对应一种数据。**只填前 6 个 Sheet 就行，后面的全部留空。**

#### Sheet 1：properties（房产表）

每一行 = 一套房子

| 列名 | 必填？ | 说明 | 正确示例 | 错误示例（为什么错） |
|------|:---:|------|---------|-------------------|
| id | ★必填 | 编号，从 1 开始：1, 2, 3... | `1` | `001`（不要加前导零） |
| property_code | ★必填 | 房产编号，格式：`FJ-0001` | `FJ-0001` | `1号房`（格式不对） |
| name | ★必填 | 房子名称，自己认识就行 | `尚景园1栋501` | （不能为空） |
| address | ★必填 | 详细地址 | `北京市朝阳区尚景园1栋501` | （不能为空） |
| status | ★必填 | 当前状态 | `空闲` | `空`（必须是这三个字之一） |
| remark | 选填 | 备注，可留空 | `朝南精装修` | （留空也行） |

**可填的状态只有 3 个：** `空闲`、`已租`、`维修中`。填别的字系统不认识。

#### Sheet 2：tenants（租客表）

每一行 = 一个租客

| 列名 | 必填？ | 说明 | 正确示例 | 错误示例（为什么错） |
|------|:---:|------|---------|-------------------|
| id_number | ★必填 | 身份证号，18位 | `320102199001011234` | `123`（位数不够） / `32010219900101123`（17位） |
| name | ★必填 | 姓名 | `张三` | （不能为空） |
| phone | ★必填 | 手机号，11位，1开头 | `13812345678` | `1234`（位数不够） / `23812345678`（不是1开头） |
| remark | 选填 | 备注 | `公司职员` | （留空也行） |

#### Sheet 3：contracts（合同表）

每一行 = 一份合同。**这是最关键的 Sheet**——它把房子和租客关联起来。

| 列名 | 必填？ | 说明 | 正确示例 | 错误示例 |
|------|:---:|------|---------|---------|
| id | ★必填 | 编号，从 1 开始 | `1` | `001` |
| contract_code | ★必填 | 合同编号，格式：`HT-0001` | `HT-0001` | `合同1` |
| property_id | ★必填 | **填 properties 表的 id**（数字） | `1` | `FJ-0001`（这是编号，不是 id） |
| tenant_id_number | ★必填 | **填 tenants 表的 id_number**（18位） | `320102199001011234` | `张三`（要填身份证号，不是名字） |
| residents_count | 选填 | 住几个人，默认 1 | `2` | `0`（至少 1 人） |
| monthly_rent | ★必填 | 月租金（元） | `2500` | `-100`（不能是负数） |
| deposit | ★必填 | 押金（元），通常=月租 | `2500` | `0`（可以，但不建议） |
| payment_method | 选填 | 付款方式，默认「月付」 | `月付` | （留空也行） |
| rent_due_day | ★必填 | 每月几号交租（1~28） | `5` | `30`（不能超过28） |
| water_fee | 选填 | 水费（元/月），不收填 0 | `60` | `-10`（不能是负数） |
| start_date | ★必填 | 合同开始日期 | `2026-01-01` | `2026年1月1日`（格式不对！） |
| end_date | ★必填 | 合同结束日期 | `2027-01-01` | `2026-01-01`（不能和开始日期一样） |
| key_count | 选填 | 给几把钥匙，默认 0 | `2` | `-1`（不能是负数） |
| status | ★必填 | 合同状态 | `待交房` | `已租`（新合同应该先填「待交房」） |
| remark | 选填 | 备注 | （留空也行） | |

> **日期格式必须是 YYYY-MM-DD**。比如 2026 年 1 月 1 日 → 写 `2026-01-01`。不要写 `2026年1月1日`、`2026/01/01` 等任何其他格式。

#### Sheet 4：contract_items（合同物品清单）

每一行 = 合同里的一件物品。一个合同可以有多件物品。

| 列名 | 必填？ | 说明 | 正确示例 |
|------|:---:|------|---------|
| id | ★必填 | 编号，从 1 开始 | `1` |
| contract_id | ★必填 | **对应 contracts 表的 id** | `1` |
| item_name | ★必填 | 物品名称 | `双人床` |
| quantity | ★必填 | 数量 | `1`（有多少写多少，至少 1） |
| sort_order | 选填 | 排序，1, 2, 3... | `1` |

#### Sheet 5：move_in_inspections（交房验收表）

每一行 = 一份合同的交房验收。一份合同只需一行。

| 列名 | 必填？ | 说明 | 正确示例 | 错误示例 |
|------|:---:|------|---------|---------|
| id | ★必填 | 编号，从 1 开始 | `1` | `001` |
| contract_id | ★必填 | **对应 contracts 表的 id** | `1` | |
| inspection_date | ★必填 | 验收日期 | `2026-01-01` | |
| meter_base_reading | ★必填 | **电表底数（入住时的电表数字）** | `5200` | `0`（除非电表真的是 0） |
| key_delivery_detail | 选填 | 钥匙交付说明，填 `{}` | `{}` | |

> **电表底数非常重要！** 以后每个月的电费都是按这个数来算的。填错了，所有电费都算错。

#### Sheet 6：move_in_inspection_items（交房物品验收表）

每一行 = 交房验收里的一件物品。对应 contract_items 里的每一件。

| 列名 | 必填？ | 说明 | 正确示例 |
|------|:---:|------|---------|
| id | ★必填 | 编号，从 1 开始 | `1` |
| inspection_id | ★必填 | **对应 move_in_inspections 表的 id** | `1` |
| item_name | ★必填 | 物品名称（和合同物品清单一致） | `双人床` |
| quantity | ★必填 | 数量 | `1` |
| status | ★必填 | `完好` 或 `有瑕疵` | `完好` |
| defect_remark | 选填 | 如果有瑕疵，描述问题 | `柜门有点松` |

#### Sheet 7~15：全部留空

以下 Sheet **不需要填，保留表头行，下面不要填数据**：
- meter_readings（抄表）
- bills（账单）
- bill_other_fees
- payments（收款）
- payment_allocations
- dunning_logs（催收）
- move_out_inspections（退房验收）
- move_out_inspection_items
- settlements（结算单）

这些数据是你在日常使用中系统自动产生的，不需要初始化时填写。

### 4.4 完整填写示例

假设你有两套房要出租，这是完整填法：

#### properties（房产）

| id | property_code | name | address | status | remark |
|----|--------------|------|---------|--------|--------|
| 1 | FJ-0001 | 尚景园1栋501 | 北京市朝阳区尚景园1栋501室 | 空闲 | 精装修 |
| 2 | FJ-0002 | 阳光花园3栋202 | 北京市海淀区阳光花园3栋202室 | 空闲 | |

#### tenants（租客）

| id_number | name | phone | remark |
|-----------|------|-------|--------|
| 320102199001011234 | 张三 | 13812345678 | 附近上班 |
| 320103199205052345 | 李四 | 13987654321 | |

#### contracts（合同）

| id | contract_code | property_id | tenant_id_number | monthly_rent | deposit | rent_due_day | water_fee | start_date | end_date | status |
|----|--------------|-------------|-----------------|-------------|---------|-------------|-----------|------------|----------|--------|

### 4.6 常见填表错误

| 你做了什么 | 后果 | 怎么办 |
|-----------|------|--------|
| 改了表头（第一行那些英文字） | 这一列数据全部丢失 | **表头一个字都别动** |
| `property_id` 填了 `FJ-0001` | 系统找不到房子 | 应该填数字 `1`，不是 `FJ-0001` |
| 日期写了 `2026年1月1日` | 导入失败 | 必须写 `2026-01-01` |
| 状态写了 `空` 或 `出租中` | 系统不认识 | 只能写 `空闲`、`已租`、`维修中` |
| 身份证号填了 17 位 / 19 位 | 导入失败 | 必须是 18 位 |
| 手机号填了 10 位 / 12 位 | 导入失败 | 必须是 11 位，1 开头 |
| Sheet 名被改了 | 这个 Sheet 的数据全部跳过 | Sheet 名不要改 |
| 电表底数忘了填 / 填了 0 | 以后抄表报错 | `meter_base_reading` 必填 |
| 结束日期填的比开始日期早 | 导入失败 | 结束日期必须晚于开始日期 |
| Excel 另存成了 `.csv` 或 `.xls` | 系统读不了 | 必须保存为 `.xlsx` |

## 5. 备份数据（防止数据丢失）

### 5.1 怎么备份

1. 打开系统首页（手机 App 点首页按钮，浏览器输入系统地址）
2. 找到快捷操作里的 **「备份数据」** 按钮（在页面中间位置，6 个按钮之一）
3. **点击它**
4. 浏览器右下角会自动下载一个文件，名叫 `landlord_backup_2026XXXX.xlsx`
5. 把这个文件**保存到你记得住的地方**（桌面、U盘、微信收藏都行）

**建议频率**：每个月收完租后备份一次，或者每次退租结算后备份一次。

### 5.2 怎么恢复（从备份还原）

操作和 4.5 的「导入 Excel」完全一样。备份文件就是 Excel 文件，直接用「恢复数据」功能导入即可。

### 5.3 换电脑怎么迁移

1. 旧电脑上：首页 → 备份数据 → 下载 Excel
2. 把 Excel 文件拷到新电脑（U盘/微信/网盘都行）
3. 新电脑上：装好系统 → 首页 → 恢复数据 → 选文件 → 导入

## 6. 常见问题解答

### Q1：系统打不开？
**答：** 确保手机或电脑已联网。如果是浏览器访问，检查地址是否正确。如果是手机 App，确认后端服务正常运行。

### Q2：逾期账单是怎么判断的？
**答：** 系统自动检查：如果一张账单的「截止日期」已经过了今天，而且还没付完，就是「逾期」。每次打开首页或账单页都会自动检查。

### Q3：电费怎么算的？有小数吗？
**答：** 电费 = `(这次电表数 - 上次电表数) × 1.2 元/度`。比如上次 1000，这次 1101，用电 101 度，电费 = 101 × 1.2 = 121.2 元。系统自动算，不需要你手动填。小数会保留（比如 121.2 元）。

### Q4：为什么不能手动把房子改成"已租"？
**答：** 房子状态是系统自动管的。签合同 → 自动变"已租"，退租 → 自动变"空闲"。你不需要手动改。

### Q5：退租后合同不见了，正常吗？
**答：** 正常。结算确认后 1 分钟，合同自动从列表删除。想查看历史数据，去「租客管理」→「已退租」→ 点那个租客进去，所有历史数据都在里面。

### Q6：退租后数据能保存多久？
**答：** 150 天（大概 5 个月）。超过 150 天后，可以去首页点「清理归档」手动删除。

### Q7：怎么恢复一个已经退租的租客？
**答：** 去「租客管理」→「已退租」→ 点那个租客 → 拉到页面底部 → 点「恢复租客」→ 确认。状态变回「在用」，可以重新签合同。

### Q8：付款后发现还有欠款？
**答：** 可能是付完款后又录入了新的抄表（电费增加了）。系统会自动把账单状态从「已付」改回「部分付」，差额会在账单详情里显示。

### Q9：系统地址是什么？
**答：** 手机 App 不需要填地址，自动连接。浏览器用户请联系管理员获取系统地址，通常类似 `https://landlord-xxx.onrender.com`。

### Q10：Excel 导入失败了怎么办？
**答：** 逐项检查：
1. 日期格式是不是 `YYYY-MM-DD`？
2. 身份证号是不是刚好 18 位？
3. 手机号是不是 11 位、1 开头？
4. property_id 填的是不是数字（不是 FJ-0001）？
5. 状态是不是只填了允许的那几个字？
6. 表头是不是一个字都没改？
7. 文件格式是不是 `.xlsx`？
